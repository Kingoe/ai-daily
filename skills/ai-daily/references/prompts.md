# AI Daily 提示词模板

## 1. 摘要生成提示词

### 中文摘要

```
你是一名专业的 AI 行业资讯编辑。请为以下新闻生成中文摘要。

要求：
1. 严格控制在 200 字以内
2. 只陈述事实，不添加主观评价
3. 以"来源："开头
4. 包含核心信息和关键细节（数字、名称等）
5. 语言简洁、客观、专业

新闻内容：
{content}

请生成中文摘要：
```

### 英文摘要

```
You are a professional AI industry editor. Please generate an English summary for the following news.

Requirements:
1. Strictly limit to 200 words
2. State facts only, no subjective opinions
3. Start with "Source: "
4. Include core information and key details (numbers, names, etc.)
5. Use concise, objective, professional language

News content:
{content}

Please generate English summary:
```

## 4. 重要性评分提示词

```
你是一名资深 AI 行业分析师。请评估以下新闻的重要性并打分（1-5 分）。

评分标准：
5 分：重大产品发布/战略级合作（如 GPT-5 发布、OpenAI 与苹果合作）
4 分：重要产品更新/大额融资（如 Claude 大版本更新、B 轮以上融资）
3 分：常规产品迭代/学术研究（如小版本更新、arXiv 论文）
2 分：行业资讯/minor 更新（如人事变动、小功能优化）
1 分：简讯/转载内容（如短消息、会议预告）

新闻内容：
{content}

请给出评分（1-5）并说明理由：
```

## 5. 去重判断提示词

```
你是一名数据去重专家。请判断以下两条新闻是否报道的是同一事件。

新闻 A：
标题：{title_a}
内容：{content_a}

新闻 B：
标题：{title_b}
内容：{content_b}

判断标准：
- 核心事件是否相同
- 关键信息（公司、产品、时间）是否一致
- 是否为同一事件的不同角度报道

请回答"是"或"否"，并说明理由：
```

## 6. 信息源可信度评估提示词

```
你是一名事实核查专家。请评估以下新闻来源的可信度。

来源：{source_name}
URL: {url}
内容类型：{content_type}（官方公告/新闻报道/自媒体/博客等）

评估维度：
1. 来源权威性（官方 > 权威媒体 > 一般媒体 > 自媒体）
2. 历史准确性
3. 是否有利益相关

请给出可信度评级（高/中/低）并说明理由：
```

## 7. 完整流水线提示词（一体化）

```
你是一名 AI 日报生成助手。请完成以下任务：

## 输入数据
{raw_news_items}

## 任务清单

1. **过滤**：排除与 AI 无关的内容
2. **去重**：合并同一事件的不同报道
3. **摘要**：为每条新闻生成中英文摘要（各≤200 字/词）
4. **标签**：为每条新闻分配 1-2 个分类标签
5. **评分**：按重要性打分（1-5 分）
6. **格式化**：输出符合 schema 的 JSON

## 输出格式
请严格按照以下 JSON Schema 输出：
{json_schema}

## 注意事项
- 中文源（量子位、机器之心）不需要英文摘要
- 英文源需要双语摘要
- 合并的来源记录在 merged_sources 数组中
- 确保所有字段都符合 schema 要求

请开始处理：
```

## 8. 日期识别提示词

```
请从以下文本中提取新闻发布日期：

文本：{text}

要求：
1. 识别明确的日期（如"2026 年 3 月 25 日"、"March 25, 2026"）
2. 识别相对日期（如"今天"、"yesterday"）并转换为绝对日期
3. 输出 ISO 8601 格式：YYYY-MM-DDTHH:MM:SSZ

如果无法识别，返回 null。

发布日期：
```
